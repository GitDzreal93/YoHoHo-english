# 卡通英语闪卡APP产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品背景
随着全球化进程加速，幼儿英语教育已成为家长关注的重点。传统的英语学习方式往往枯燥乏味，难以激发3-6岁儿童的学习兴趣。本项目旨在利用现有的丰富卡通图片资源（287+动物主题，22个分类），打造一款寓教于乐的英语学习应用。

### 1.2 产品名称
**Kids Cartoon Flashcards - 卡通英语闪卡**

### 1.3 产品定位
面向3-6岁非英语母语国家儿童的英语学习应用，通过卡通闪卡和趣味游戏帮助儿童建立英语词汇基础。

### 1.4 目标用户
- **主要用户群体**：3-6岁儿童
  - 3-4岁：认知启蒙阶段，以图像和声音为主
  - 4-5岁：词汇积累阶段，开始简单的拼写练习
  - 5-6岁：能力提升阶段，可以进行更复杂的语言互动
- **次要用户群体**：家长、幼儿教师
- **市场定位**：非英语母语国家的早期英语教育市场

### 1.5 核心价值主张
通过生动有趣的卡通图片和多感官互动体验，让幼儿在玩乐中自然习得英语词汇，培养英语学习兴趣。

### 1.6 产品目标
- **短期目标**：6个月内完成MVP版本开发，包含核心闪卡功能和2个游戏
- **中期目标**：1年内用户下载量达到10万+，付费转化率15%+
- **长期目标**：成为亚太地区领先的幼儿英语学习品牌

## 2. 市场分析

### 2.1 市场需求
- **市场规模**：全球幼儿英语教育市场价值超过100亿美元
- **增长趋势**：年复合增长率12%+
- **用户痛点**：
  - 传统学习方法枯燥，儿童注意力难集中
  - 缺乏系统性的幼儿英语学习内容
  - 家长缺乏有效的辅导工具
  - 现有应用质量参差不齐

### 2.3 差异化优势
- **内容优势**：基于3000+张精选卡通图片，质量统一
- **技术优势**：智能推荐算法，个性化学习路径

- **本土化优势**：针对非英语母语国家用户设计

## 3. 功能需求

### 3.1 核心功能模块

#### 3.1.1 闪卡学习模块

**基础闪卡浏览**
- **卡片展示**：
  - 正面：高清卡通图片 + 英文单词
  - 背面：中文翻译 + 音频播放按钮
  - 翻转动画：3D翻转效果，增强互动性
- **音频功能**：
  - 标准美式发音播放
  - 中文翻译播放
  - 语速调节：0.8x、1.0x、1.2x
  - 循环播放功能
- **分类浏览**：
  - 22个主题分类（基于现有categories.json）
  - 推荐：按年龄和难度推荐分类
  - 收藏：个人喜爱的卡片收藏夹

**学习模式**
- **浏览模式**：
  - 自由选择分类和词汇
  - 顺序浏览或随机浏览
  - 支持搜索功能
- **学习模式**：
  - 基于遗忘曲线的复习提醒
  - 智能推荐新词汇
  - 学习进度追踪
- **复习模式**：
  - 专项复习难记词汇
  - 错题集功能
  - 强化训练模式

#### 3.1.2 趣味游戏模块

**游戏1：声音寻宝 (Sound Treasure Hunt)**

*游戏机制*：
1. 播放英文单词标准发音
2. 屏幕显示2-6张卡通图片
3. 儿童点击对应的图片
4. 答对获得星星奖励，答错给予鼓励提示

*难度设计*：
- **入门级**：2选1，常见动物词汇
- **基础级**：3选1，日常物品词汇
- **进阶级**：4选1，抽象概念词汇
- **挑战级**：6选1，复杂词汇

*学习目标*：
- 听觉识别能力训练
- 词汇音义联系建立
- 反应力训练

**游戏2：魔法拼图 (Magic Puzzle)**

*游戏机制*：
1. 卡通图片被分割成2-4块拼图
2. 儿童拖拽拼图到正确位置
3. 完成拼图后播放单词发音和动画
4. 收集完整图片集解锁新内容

*难度设计*：
- **简单级**：2块拼图，明显分割线
- **中等级**：3块拼图，不规则分割
- **困难级**：4块拼图，复杂形状

*学习目标*：
- 形状认知能力
- 手眼协调能力
- 词汇图像记忆

**游戏3：彩虹泡泡 (Rainbow Bubbles)**

*游戏机制*：
1. 屏幕飘出彩色泡泡，每个泡泡显示一个字母
2. 儿童按正确顺序点击泡泡
3. 完成拼写后显示对应图片和发音
4. 连续正确获得combo奖励

*难度设计*：
- **3字母单词**：Cat, Dog, Sun等简单词汇
- **4字母单词**：Bird, Fish, Duck等基础词汇
- **5字母单词**：Tiger, Horse, Panda等进阶词汇

*学习目标*：
- 字母识别能力
- 基础拼写训练
- 字母顺序概念

**游戏4：动物音乐盒 (Animal Music Box)**

*游戏机制*：
1. 卡通动物形象会"唱歌"（播放单词发音）
2. 儿童跟着模仿发音
3. 系统进行语音识别和评分
4. 评分标准：准确度、流利度、语调

*难度设计*：
- **单音节词**：Cat, Dog, Pig
- **双音节词**：Tiger, Rabbit, Monkey
- **多音节词**：Elephant, Crocodile

*学习目标*：
- 发音准确性训练
- 口语表达能力
- 语音模仿能力

**游戏5：记忆翻翻乐 (Memory Flip)**

*游戏机制*：
1. 经典记忆配对游戏
2. 卡片分为图片卡和单词卡
3. 翻开找到匹配的图片-单词对
4. 记录翻牌次数和时间

*难度设计*：
- **入门级**：4张卡片（2对），2x2布局
- **基础级**：6张卡片（3对），2x3布局
- **进阶级**：8张卡片（4对），2x4布局
- **挑战级**：12张卡片（6对），3x4布局

*学习目标*：
- 记忆力训练
- 词汇图像联系
- 专注力培养

**游戏6：单词小画家 (Word Artist)**

*游戏机制*：
1. 提供简化的卡通轮廓
2. 儿童选择颜色填充
3. 完成后显示英文单词和发音
4. 作品可保存和分享

*难度设计*：
- **填色模式**：简单轮廓，大片区域
- **描边模式**：需要沿着轮廓描画
- **自由创作**：提供基础元素自由组合

*学习目标*：
- 创意表达能力
- 颜色认知
- 词汇学习

#### 3.1.3 学习进度管理系统

**学习统计**
- **每日数据**：
  - 学习时长统计
  - 新学词汇数量
  - 复习词汇数量

  - 学习建议

**成就系统**
- **等级体系**：
  - 初学者 (Beginner)：0-50词汇

  - 学习者 (Learner)：151-300词汇
  - 专家 (Expert)：301-500词汇
  - 大师 (Master)：500+词汇
- **成就徽章**：
  - 连续学习徽章（7天、30天、100天）
  - 游戏高手徽章（各游戏通关）
  - 词汇收藏家徽章（掌握特定分类词汇）
  - 发音达人徽章（语音识别高分）

**个性化推荐**
- **智能推荐算法**：
  - 基于学习历史推荐难度适中的词汇
  - 根据错误模式推荐强化练习
  - 考虑年龄和认知水平
- **学习路径定制**：
  - 个性化学习计划
  - 灵活调整学习节奏
  - 多样化学习方式组合

#### 3.1.4 家长功能模块

**进度监控**
- **实时仪表板**：
  - 今日学习情况
  - 本周进度概览
  - 能力发展雷达图
- **详细报告**：
  - 每日学习日志

  - 游戏表现分析

**内容管理**
- **分类选择**：
  - 可选择特定分类进行学习

- **难度调节**：
  - 根据孩子水平调整难度

**时间控制**
- **使用时长管理**：

  - 设置可用时间段

#### 3.2.2 音频处理需求

**音频格式标准**
- **格式支持**：WAV, MP3, AAC
- **音质要求**：44.1kHz, 16-bit, 立体声
- **文件大小**：单词汇音频文件<50KB
- **加载时间**：音频文件加载<0.5秒

**音频功能**
- **播放控制**：播放、暂停、停止、循环
- **语速调节**：0.8x, 1.0x, 1.2x
- **音量控制**：独立音量调节
- **音频缓存**：智能预加载机制

#### 3.2.3 图像处理需求

**图像标准**
- **支持格式**：PNG, JPG, WebP
- **分辨率要求**：
  - 手机：适配1080p屏幕
  - 平板：适配2K屏幕
  - 高清：支持4K屏幕
- **文件大小**：单张图片<200KB
- **加载优化**：渐进式加载，占位图显示

**图像功能**
- **缩放适配**：不同屏幕尺寸自适应
- **动画效果**：淡入淡出、滑动、缩放
- **缓存策略**：LRU缓存算法
- **压缩优化**：WebP格式，体积减少30%

## 4. 用户体验设计

### 4.1 界面设计原则

#### 4.1.1 儿童友好设计
- **色彩设计**：
  - 主色调：明亮、温暖的颜色
  - 对比度：高对比度，便于识别
  - 色彩心理学：运用色彩心理学原理
- **字体设计**：
  - 字体选择：圆润、友好的无衬线字体
  - 字号大小：适合儿童阅读的大字号
  - 字重：中等字重，清晰易读
- **图标设计**：
  - 风格统一：扁平化、卡通风格
  - 识别度高：直观的图标设计
  - 尺寸适中：适合儿童手指点击

#### 4.1.2 交互设计
- **操作简化**：
  - 支持点击、拖拽、滑动等基础手势
  - 避免复杂的多点触控操作
  - 提供操作引导和提示
- **反馈机制**：
  - 即时视觉反馈：按钮点击效果、状态变化
  - 音频反馈：操作音效、鼓励语音
  - 触觉反馈：振动反馈（可选）
- **容错设计**：
  - 允许操作失误，不严厉惩罚
  - 提供重新尝试的机会
  - 错误操作友好提示

### 4.2 用户体验流程

#### 4.2.1 首次使用流程
1. **欢迎界面**：友好的卡通角色欢迎
2. **年龄选择**：选择孩子年龄段（3-4岁、4-5岁、5-6岁）
3. **兴趣测试**：通过图片选择了解孩子兴趣
4. **家长设置**：设置家长控制和使用时间
5. **开始学习**：引导进入第一个学习模块

#### 4.2.2 日常学习流程
1. **登录界面**：孩子头像选择
2. **今日推荐**：基于进度的个性化推荐
3. **学习选择**：闪卡学习或游戏选择
4. **学习过程**：沉浸式学习体验
5. **学习总结**：今日成就展示
6. **奖励系统**：星星、徽章收集

### 4.3 无障碍设计

#### 4.3.1 视觉无障碍
- **色盲友好**：不仅依赖颜色区分信息

#### 4.3.2 听觉无障碍

### 5.2 难度分级体系

#### 5.2.1 初级（3-4岁）
**词汇特征**：
- **长度**：3-5个字母
- **类型**：具体名词为主
- **发音**：单音节或简单双音节
- **认知**：日常生活中常见物品

**代表词汇**：
- 动物：cat, dog, pig, cow, bird
- 颜色：red, blue, sun, moon
- 数字：one, two, three, four
- 食物：egg, milk, cake, apple

**学习目标**：
- 建立英语学习兴趣
- 掌握基础词汇发音
- 培养语音感知能力

#### 5.2.2 中级（4-5岁）
**词汇特征**：
- **长度**：5-8个字母
- **类型**：包含抽象概念
- **发音**：多音节词汇
- **认知**：需要一定理解能力

**代表词汇**：
- 动物：rabbit, monkey, tiger, elephant

def calculate_pronunciation_score(user_audio, target_audio):
    # 音频特征提取
    user_features = extract_features(user_audio)
    target_features = extract_features(target_audio)

    # 相似度计算
    similarity = cosine_similarity(user_features, target_features)

    # 时间对齐评分
    timing_score = calculate_timing_score(user_audio, target_audio)

    # 音调评分
    intonation_score = calculate_intonation_score(user_audio, target_audio)

    # 综合评分
    final_score = (similarity * 0.5 +
                  timing_score * 0.3 +
                  intonation_score * 0.2)

    return final_score
```

class AdaptiveDifficulty:
    def __init__(self):
        self.user_skill_level = 0.5  # 0-1
        self.recent_performance = []
        self.difficulty_multiplier = 1.0

    def update_difficulty(self, game_result):
        # 记录最近表现
        self.recent_performance.append(game_result)
        if len(self.recent_performance) > 10:
            self.recent_performance.pop(0)

        # 计算平均表现
        avg_performance = sum(self.recent_performance) / len(self.recent_performance)

        # 调整难度
        if avg_performance > 0.8:  # 表现优秀，增加难度
            self.difficulty_multiplier *= 1.1
        elif avg_performance < 0.4:  # 表现不佳，降低难度
            self.difficulty_multiplier *= 0.9

        # 限制难度范围
        self.difficulty_multiplier = max(0.5, min(2.0, self.difficulty_multiplier))
```
